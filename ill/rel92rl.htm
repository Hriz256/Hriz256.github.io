Adobe Acrobat is required to access Orcad online manuals

Adobe Acrobat is required to access Orcad family release 9.2 online manuals. If you do not have Adobe Acrobat installed on your system, you are given the option to install it during the Release 9.2 installation.

If you already have a previous version (3.x) of Adobe Acrobat installed on your system, you are not given the option to upgrade during the Release 9.2 installation (because you are forced to reboot your system after uninstalling the previous version of Adobe Acrobat), but it is still important that you install the latest version of Adobe Acrobat. If you do not install Acrobat 4.05, you may not be able to perform searches across the various Release 9.2 online manuals.

If you have an older version of Adobe Acrobat installed on your system, follow the steps below.

  1. First, you must uninstall the previous version of Acrobat. From the Start menu, point to Programs, point to Adobe Acrobat, then choose Uninstall Acrobat n.nn.

Alternatively, you can uninstall Acrobat using the Add/Remove Programs option from your Windows Control Panel. (From the Start menu, point to Settings, point to Control Panel, double-click on the Add/Remove Programs icon, choose Adobe Acrobat n.nn from the list of programs, then click Add/Remove.)

  1. After you have uninstalled the previous version of Acrobat, you are ready to install Acrobat 4.05. Insert your Orcad family release 9.2 CD in your CD drive and use Windows Explorer to access the Adobe directory, then double-click on RS405ENG.EXE to start the Acrobat installation.

 
Installing FLEXlm as a service under Windows NT 4.0

There are several advantages to having the Cadence FLEXlm License Manager installed as a service under Windows NT. As a service, the license manager is activated upon powering-up the server whether a user has logged on to the machine or not. This is especially important in secure or remote server installations where the users may not have access to the server, servers that are subject to power failures, or any time automatic activation of the license manager is required. Installing FLEXlm as a service is a straightforward process, and can be accomplished easily by a qualified system administrator.

NOTE: This procedure is performed after the license manager has been installed using SETUP.EXE on the Orcad family release 9.2 CDROM.

  1. If you do not have other programs using FLEXlm as a service, then, on the server, copy the FLEXLM.CPL and LMGR325C.DLL files from the the C:\ Program Files\Orcad \LIC_MNGR directory to the C:\WINNT\SYSTEM32 directory. Otherwise, continue to step 2

  2. Double-click on the FLEXLM.CPL icon in the Windows control panel to display the FLEXlm License Manager dialog box.

  3. Choose the Setup tab. Enter the data for the location of the FLEXlm files.

    1. Service Name - Use the default unless you have more than one FLEXlm license manager running as a service.

    2. LMGRD.EXE - Using the Browse button, choose the C:\ Program Files\Orcad \LIC_MNGR\ directory, or the directory where you have installed the Cadence license manager software.

    3. License File - Using the Browse button, choose the C:\ Program Files\Orcad \LIC_MNGR\ directory, or the directory where you have installed the Cadence license manager software.

    4. Log File - Using the Browse button, choose the C:\ Program Files\Orcad \LIC_MNGR\ directory, or the directory where you have installed the Cadence license manager software.

    5. Select the box Use NT Services check box to enable the software as a service.

    6. Select the Start Server at Power-Up check box to automatically start the service on power up.

  4. Choose the Control tab. Stop the server if it is already started.

  5. In the Windows NT control panel, double-click the Services icon.

    1. Find the FLEXlm License Manager service. If you renamed the service under step 3A, then find that service name.

    2. Select the service, and click Startup. Verify that the service is set up as Automatic and as a System Account log on, then close the window dialog box.

    3. Click Start. The status of the service should now indicate that it has been started. Click Close.

  6. Remove the Cadence License Manager icon from any Startup folders. Be sure to check ALL startup folders. Your server will function correctly with more than one copy of the Cadence FLEXlm license manager installed.

The FLEXlm license manager will now be installed as a Windows NT service. If you wish, you may test the installation by rebooting the server, and starting the Cadence software on a client machine.

 

Capture installed as a remote client

When you invoke Capture installed as a remote client, you may get a message that the CAPTURE.INI file cannot be opened for writing. To correct this problem, create a Capture directory in Program Files in your local directory structure so that it is the same as the path in the error message. The next time you start Capture, a CAPTURE.INI file is created.

General protection faults occur while using NetHASP Monitor

Problem

NetHASP Monitor does not work under Windows 2000.

During compatibility testing, we determined that the NetHASP Monitor program does not work under Windows 2000. We were unable to obtain a version of this software that works with Windows 2000 in time for this release.

Workaround

If you need this software, please check with Cadence Customer Support to obtain the latest release of Monitor.
